## 1. 클라이언트 측

| 주체 | 역할 |
|------|------|
| **창신** | 화장품 용기 제조사. 시스템 도입 주체 |
| **모니터 딜로이트** | 창신 측에 들어온 컨설팅 업체. PPT v6 워크플로우 작성 |
| **서주현 팀장** (모니터 딜로이트) | 본인의 **직접 커뮤니케이션 카운터파트**. 미팅 진행, Action Items 책임 일부 |

## 2. 개발사 측 — 그룹사 구조

| 주체 | 역할 |
|------|------|
| **팀스파르타** | **모기업** |
| **엄영태 팀장** (팀스파르타 PM) | **ERP 액세스를 받은 인물**. ERP 액세스 라인 = 창신 → 엄영태 → 본인. 정확한 PM role 은 미확인 (이 프로젝트 한정인지 그룹 차원인지) |
| **스파르타빌더스** (자회사 · **구 위플래닛**) | 팀스파르타 자회사. **실제 개발 담당** |
| **본인 (Ethan Yoo)** | AX-2 담당 개발자 |
| **AX1·AX3 담당자** | 본인과 별개. 구체 인물 미상 |

## 4. ⚠ 핵심 주의 사항

### 사명 변경 (위플래닛 → 스파르타빌더스)

> codebase 의 **"위플래닛 표준"** (`@Auth`, `@User`, JWT 페이로드, S3 URL 컨벤션, `@ApiProperty` 통합 등) = **현재 스파르타빌더스 표준**.
> 동일한 회사를 가리키는 옛 이름이므로 헷갈리지 말 것.
> 임시 도메인 `*.dev.weplanet.co.kr` 도 동일한 회사의 옛 도메인.

### 회의 녹음 AI 인식 오류

> 회의 녹음 AI 가 **"회장님"으로 인식한 표현은 대부분 "엄영태 팀장"** 을 가리킴.
> 회의록 활용 시 항상 사실 정정 필요.

```
ERP 액세스: 창신 → (엄영태 팀장) → 본인           ← 한 단계 거침
요구사항:   창신 ↔ 서주현 팀장 ↔ 본인              ← 직접 커뮤니케이션
도메인 결정: 창신/모니터 딜로이트 → 본인           ← 모니터 딜로이트가 클라 측 정리 후 전달
```
