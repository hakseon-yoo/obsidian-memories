# Module 2 · 생산 계획 관리

> 상위 문서: [[00 - AX-2 쉬운 설명서 (Index)]]

> [!summary] 한 줄로 말하면
> 생산 계획 수립 → 외주처 컨펌 → 확정 → 매일 진행상황 추적 → 납기/품질 이슈 대응까지 **자동화**.

- **관련 주체**: 생산관리팀 ↔ 외주처(다수)
- **핵심 요소**: 외주 발주 자동화 / 매일 확정 계획 전달 / 납기 임박 컨펌·품질 Go/No Go

---

## 실제 현장 장면 (Before)

- 생산관리팀이 엑셀에 SKU·수량·납기일을 적어 카톡으로 외주처에 배포.
- 외주처 회신을 일일이 받아 ERP에 수기 입력 — 거의 하루 소요.
- 납기 임박 건은 ERP가 빨개져야 알아차리고 뒤늦게 컨펌.
- 품질 애매한 건도 카톡으로 사진 돌려 판정 요청 → 자주 누락.

## AX-2 적용 후 (After)

- 수주 확정 즉시 외주처별 발주서 자동 생성 및 전송.
- 외주처는 납기 가능 여부만 시스템에 체크 → ERP에 확정계획 자동 반영.
- 임박한 건은 D-n일 자동 컨펌 요청 · 당일 납품 여부 · 납품 수량·거래명세표 업로드.
- WIP 품질 이슈는 사진·동영상 첨부 후 품질팀이 Go/No Go 판정 + 방문 일정 기록.

---

## 개발팀이 구현해야 할 기능

- [ ] (5)/(6)/(8) 생산 계획 전달-컨펌-확정의 ERP 자동 입력 파이프라인
- [ ] (13) 매일 확정 생산 계획 전달 — 임박 건 별도 알람, 명일/당일 납기 컨펌 요청
- [ ] (14) 납기 조정 요청 기능: 외주처가 못 지키겠다 선택 시 즉시 (17) 스케줄 재조정 트리거
- [ ] 품질 판정 요청 기능: 상세 · 사진/동영상 첨부, 품질팀 Go/No Go + 방문 필요 시 일정 기입
- [ ] 납품 당일 마감 시점: 납품 수량 · 거래명세표 · 검수 결과 · 미수령 시 전공정/생관 노티

---

> [!warning] 개발 시 반드시 체크
> - 공정별 "며칠 전 컨펌" 기준 실무진 확인 필요
> - 수입검사 미진행 외주처 많음 — 노티 정책 현실성 검토
> - 전공정↔후공정 수량 불일치 자동 노티 로직 정의 필요
> - 품질팀 독단 판정 케이스 방지 장치 설계
