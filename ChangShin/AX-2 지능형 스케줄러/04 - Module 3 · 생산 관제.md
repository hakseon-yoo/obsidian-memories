# Module 3 · 생산 관제

> 상위 문서: [[00 - AX-2 쉬운 설명서 (Index)]]

> [!summary] 한 줄로 말하면
> 확정된 생산 계획을 **매일 아침 생산팀(창신 자체 + 외주처)에 작업 지시**로 내려보내고, 실행 중 발생하는 **납기 조정·품질 판정 요청**을 받아 스케줄링 모듈로 되돌리는 모듈.

- **관련 주체**: 생산팀(창신 자체 생산팀 + 외주처) · 생산관리팀 · 품질팀
- **PPT 워크플로우 매핑**: (13) 매일 확정 생산 계획 전달, (14) 확정 생산 계획 확인 및 준수 불가로 인한 납기일 조정 요청 및 품질 판정 요청
- **핵심 요소**: 매일 작업 지시 발송 / 실행 중 이슈 즉시 시스템 입력 / 스케줄링 (M2) 으로 회귀 트리거

---

## 실제 현장 장면 (Before)

- 생산관리팀이 매일 아침 엑셀 + 카톡으로 외주처/생산팀에 그날 작업을 전달.
- 못 지킬 것 같으면 외주처가 카톡으로 "이 건 못 합니다" 전달 → **카톡 누락 시 일정 깨짐 발견이 늦어짐**.
- WIP 가 입고됐을 때 불량 의심이지만 결정을 못해 **품질팀에 카톡으로 사진 보내고 답을 기다림** — 그 사이 생산은 멈춤.
- 품질팀 독단 판정 후 통보 누락된 사례 발생 (사출→후공정 20일 지연 사례).

## AX-2 적용 후 (After)

- 매일 아침 시스템이 그날의 확정 작업을 생산팀(창신 + 외주처) 화면에 자동 표시.
- 임박 건은 별도 강조 + 컨펌 요청 자동 발송.
- 못 지킬 시 **납기 조정 요청** 버튼 → 즉시 스케줄링 모듈 (17) 재조정 트리거.
- WIP 불량 의심 시 **품질 판정 요청 (Go/Stop)** 버튼 → 사진/동영상 첨부, 품질팀 즉시 확인 + 방문 일정 자동 노티.

---

## 개발팀이 구현해야 할 기능

- [ ] (13) 매일 확정 생산 계획 발송 — 외주처별 / 생산팀별 분리
- [ ] 임박 건 (D-X 기준) 자동 컨펌 요청 발송 — 공정별 D-day 기준은 미확정 (P0 #6)
- [ ] 작일/당일 납기 가능 여부 컨펌 요청 (생산관리·자재팀 별도 노티)
- [ ] 납기일 당일 업무 종료 시 납품 진행 여부 + 납품 수량 + 거래명세표 첨부 (Module 4 부속 영역과 연동)
- [ ] 수령 업체 입고 수량 + 수입검사 컨펌 요청 (대부분 외주처에서 미진행 — 현실성 별도 확인)
- [ ] 전공정/후공정 수량 불일치 자동 노티 (임계값 P1 #9)
- [ ] (14) **납기 조정 요청** 화면 (사유 + 조정 일자 + 첨부) → 스케줄링 모듈 (17) 트리거
- [ ] (14) **품질 판정 요청 (Go/Stop)** 화면 — 사진/동영상 첨부 필수, 품질팀이 응답 + 방문 일정

## 다른 모듈과의 경계

| 경계 | 책임 |
|------|------|
| **M2 스케줄링** | M3 가 (14) 요청을 보내면 M2 가 (17) 재조정 — M3 는 요청만, 일정 결정은 M2 |
| **M4 물류** | (13) 에서 매일 작업 지시 발송 시 **물류 슬롯 정보도 함께 전달** — 물류 슬롯 자체는 M4 책임 |
| **부속 · 구매·정산** | 납품 당일 거래명세표 첨부는 M3 화면에서 하지만, 정산 처리는 부속 영역 |
| **품질팀 액션** | (18) 품질 판정 처리는 품질팀 swimlane — M3 는 요청 전달만, 판정 결정은 품질팀 |
