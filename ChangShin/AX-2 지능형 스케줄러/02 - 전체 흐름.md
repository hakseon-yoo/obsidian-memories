## 1부 · 주문 받기부터 생산 계획 확정까지

| STEP | 단계 | 내용 |
|------|------|------|
| **1** | 수주 접수 | 영업팀이 주문을 받아 ERP에 입력 (SKU · 수량 · 원하는 납기일) |
| **2** | 납기일 예측 | AX-2가 현재 공장 상황을 보고 "이 날까진 가능"이라고 자동 계산 |
| **3** | 생산관리팀 확정 | 수주 확정 → 외주처 배정 (신제품은 반드시 조립처 지정) |
| **4** | 생산 계획 전달 | 외주처에 "○월○일까지 이 수량 만들어 줄 수 있나요?" 자동 요청 |
| **5** | 외주처 컨펌 → 확정 | 외주처 회신 취합 → 생산 계획 최종 확정 → ERP 자동 반영 |

> [!important] 이 단계에서 개발팀이 꼭 기억할 것
> - **기존 수작업**: 엑셀로 일일이 정리 → 카톡으로 외주처에 전송 → 회신 받아 다시 ERP에 수기 입력
> - **AX-2 목표**: 수주만 들어오면 예상 납기일 자동 계산, 외주처 자동 발주/회신까지 시스템 한 방
> - **데이터 전제**: SKU 마스터에 "어느 외주처가 만드는지" 등 메타데이터가 깨끗해야 가동 가능
> - **신제품 케이스**: 개발팀 T0 · T1 · T2 일자 기입 → 금형업체 컨펌 → 그 이후 일반 생산 프로세스와 동일
> - **재생산 케이스**: 원 수주번호 하위번호로 생성, 품질·구매와 공제 로직 연동 (별도 확인 필요)

## 2부 · 매일 진행 관리 · 물류 · 정산까지

| STEP | 단계 | 내용 |
|------|------|------|
| **6** | 매일 확정 계획 전달 | 각 외주처에 오늘의 발주 현황 + 임박한 건 컨펌 요청 자동 발송 |
| **7** | 납기 조정 · 품질 판정 | 못 지킬 땐 즉시 조정 요청 · WIP 불량 애매하면 품질팀에 즉시 판정 요청 |
| **8** | 물류 슬롯 자동 생성 | 확정된 일정 기반으로 배차/창고예약 빈자리(슬롯) 자동 생성 |
| **9** | 배차 · 입고 예약 | 자재팀이 트럭 번호 · ODM 창고 예약 · 거래명세표 등 필수 정보 기입 |
| **10** | 정산 | 수주 · 납품 · 공제 내역 자동 취합 · 외주/생관/구매 컨펌 후 정산 완료 |

> [!warning] 현장에서 실제로 일어나는 일 (왜 AX-2가 필요한가)
> - **진도 관리 수작업**: 외주처 진도 엑셀 → 회신 → ERP 수기 재입력에 매일 반나절 이상 소요
> - **놓치는 일정**: 카톡으로 일정 조정/품질 판정 요청 주고받다가 빠지는 건이 발생
> - **뒤늦은 품질 이슈**: WIP 입고 후 검수 안 된 채 생산 직전에 열어보고 "이거 불량" 뒤늦게 발견
> - **창고 미예약 사고**: ODM 창고에 입고 예약 안 하면 받아주지 않음 — 월 몇 건 납기 지연 발생
> - **정산 누락**: 거래명세표 누락/생관 체크 누락으로 완료 처리 안 된 건이 월 5~6회 발생
