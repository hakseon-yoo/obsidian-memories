# Module 1 · 납기일 예측

> 상위 문서: [[00 - AX-2 쉬운 설명서 (Index)]]

> [!summary] 한 줄로 말하면
> 수주가 들어오면 현재 공장 캐파 · 기수주 계획을 보고 **예상 납기일을 자동 계산**해 알려주는 모듈.

- **관련 주체**: 영업팀 수주 → 생산관리팀이 즉시 활용
- **핵심 요소**: SKU 메타데이터 정합성 / 기수주·생산계획 반영 / 지연일 자동 계산

---

## 실제 현장 장면 (Before)

- 영업이 수주를 받으면 생산관리팀이 엑셀로 기존 계획을 펼쳐본다.
- 외주처 상황을 전화·카톡으로 물어본 뒤에야 "언제쯤 됩니다"를 대답.
- SKU 마스터가 낡아서 실제 생산처와 다른 경우가 잦다.
- 신제품은 조립처 지정이 빠져 시스템에 입력되지 않는다.

## AX-2 적용 후 (After)

- 수주 입력 즉시 "SKU A는 4주차까지 기수주 꽉 참, 그 이후 +n일 지연" 자동 회신.
- SKU 마스터와 기수주 계획을 실시간 참조.
- 신제품 케이스는 생산관리가 조립처를 반드시 지정하도록 강제.
- 영업이 외주처를 잘못 입력해도 즉시 드롭다운에서 조정 가능.

---

## 개발팀이 구현해야 할 기능

- [ ] 수주 입력 폼: SKU · 수량 · 희망납기 입력 시 예상 납기 즉시 반환
- [ ] SKU 마스터 · 기수주 · 내부 생산계획 연동 (실시간 참조)
- [ ] 신제품 플로우: 개발팀 T0/T1/T2 일자 입력 → 금형업체 컨펌 → 일반 생산 플로우 진입
- [ ] 재생산 플로우: 원수주번호의 하위번호 자동 생성 · 스케줄 타이트 적용
- [ ] 외주처 변경 시 ERP/SKU 마스터 메타 자동 동기화 (권한 체크 포함)

## 수주 시나리오 분기 (클라이언트 확정 — 2026-04-30)

> 출처: [[Meetings/2026-04-30 - 클라이언트 답변 - 수주 시나리오 · 납기 로직]] (서주현 팀장 · 모니터 딜로이트)

수주 입력 화면 진입 시 **두 시나리오 중 하나를 선택**하고, 그에 따라 입력 폼이 분기된다.

| 시나리오 | 입력 | 흐름 차이 |
|----------|------|-----------|
| **A. 기제품 수주** | 자재코드(SKU) + 수량 | 일반 생산 플로우. **후가공만 변경되는 신제품도 이 흐름** 으로 동일 처리 |
| **B. 신제품 수주** | "가장 유사한 제품"의 자재코드 + 수량 | **금형 설계 + 개발 일정** 이 추가됨. 자재코드는 유사 제품 참조용 |

화면 UX:

1. STEP 1 · 시나리오 선택 (기제품 / 신제품)
2. STEP 2 · 자재코드 검색·선택 + 수량 입력
3. STEP 3 · (B 인 경우) 신제품 메타(금형 소재/난이도/크기 등) 추가 입력
4. STEP 4 · 예상 납기일 응답 확인 → 수주 확정

## 예상 납기일 로직 (구현 필요 — 변수 정리 진행 중)

> 출처: [[Meetings/2026-04-30 - 클라이언트 답변 - 수주 시나리오 · 납기 로직]]

### 신제품 (금형 개발)

- **평균**: 약 **45일**
- **변동 변수** (가장 큰 영향순)
  1. 업체별 capa (금형 개발) ← 제일 중요
  2. 금형 난이도
  3. 금형 소재
  4. 금형 크기

### 기제품

- 부자재·공정별 평균 소요 시간이 **생산관리팀장 머릿속**에 있음
- 클라이언트(서주현 팀장) 가 **추가 인터뷰** 후 변수 정리 공유 예정
- 변수 도착 전엔 알고리즘 본체 구현 보류, 결과 저장 엔티티/인터페이스 골격만 작성

### 시스템 전제

- **외주업체 생산계획 직접 수신 불가** → 시스템 내 **기수주 점유율** 기반으로 일정 지연 추정
- 따라서 다음 마스터/집계가 필요:
  - 공정 마스터 (사출 / 후가공 / 도장 / 조립 등)
  - 외주처 마스터 (공정별 capa, 금형 개발 capa)
  - SKU ↔ 공정 ↔ 외주처 매핑
  - 공정별 평균 소요 시간 (변수: 소재 / 난이도 / 크기 / 업체)
  - 현재 기수주 점유 상태 (실시간 또는 배치 집계)

---

> [!warning] 개발 시 반드시 체크
> - SKU 마스터 데이터 클렌징이 **선결 조건**
> - 초기엔 실시간 피딩 없이 **배치 기반**으로 시작
> - 데이터 축적 후 ML 기반 고도화는 2단계로 분리
> - 외주처 기입 권한/책임 부서 확정 필요
