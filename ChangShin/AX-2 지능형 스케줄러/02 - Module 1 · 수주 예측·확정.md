# Module 1 · 수주 예측·확정

> 상위 문서: [[00 - AX-2 쉬운 설명서 (Index)]]

> [!summary] 한 줄로 말하면
> 영업팀이 **수주를 입력**하면 현재 공장 캐파·기수주를 보고 **예상 납기일을 자동 산출**한 뒤, 영업이 **수주를 확정**해 다음 모듈(스케줄링)로 넘기는 영업팀 전용 모듈.

- **관련 주체**: 영업팀
- **PPT 워크플로우 매핑**: (1) 수주 시나리오 입력 (SKU·물량·배송지) · (2) 예상 납기일 · (3) 수주 확정
- **핵심 요소**: 시나리오 분기 (기제품/신제품/재생산) / SKU 메타데이터 정합성 / 기수주·생산계획 반영 / 지연일 자동 계산 / 확정 액션 → M2 스케줄링으로 이관

---

## 실제 현장 장면 (Before)

- 영업이 수주를 받으면 생산관리팀이 엑셀로 기존 계획을 펼쳐본다.
- 외주처 상황을 전화·카톡으로 물어본 뒤에야 "언제쯤 됩니다"를 대답.
- SKU 마스터가 낡아서 실제 생산처와 다른 경우가 잦다.
- 신제품은 조립처 지정이 빠져 시스템에 입력되지 않는다.

## AX-2 적용 후 (After)

- 수주 입력 즉시 "SKU A는 4주차까지 기수주 꽉 참, 그 이후 +n일 지연" 자동 회신.
- SKU 마스터와 기수주 계획을 실시간 참조.
- 신제품 케이스는 생산관리가 조립처를 반드시 지정하도록 강제.
- 영업이 외주처를 잘못 입력해도 즉시 드롭다운에서 조정 가능.

---

## 개발팀이 구현해야 할 기능

- [ ] 수주 입력 폼: SKU · 수량 · 희망납기 입력 시 예상 납기 즉시 반환
- [ ] SKU 마스터 · 기수주 · 내부 생산계획 연동 (실시간 참조)
- [ ] 신제품 플로우: 개발팀 T0/T1/T2 일자 입력 → 금형업체 컨펌 → 일반 생산 플로우 진입
- [ ] 재생산 플로우: 원수주번호의 하위번호 자동 생성 · 스케줄 타이트 적용
- [ ] 외주처 변경 시 ERP/SKU 마스터 메타 자동 동기화 (권한 체크 포함)

## 수주 시나리오 분기 (클라이언트 확정 — 2026-04-30)

> 출처: [[Meetings/2026-04-30 - 클라이언트 답변 - 수주 시나리오 · 납기 로직]] (서주현 팀장 · 모니터 딜로이트)

수주 입력 화면 진입 시 **두 시나리오 중 하나를 선택**하고, 그에 따라 입력 폼이 분기된다.

| 시나리오 | 입력 | 흐름 차이 |
|----------|------|-----------|
| **A. 기제품 수주** | 자재코드(SKU) + 수량 | 일반 생산 플로우. **후가공만 변경되는 신제품도 이 흐름** 으로 동일 처리 |
| **B. 신제품 수주** | "가장 유사한 제품"의 자재코드 + 수량 | **금형 설계 + 개발 일정** 이 추가됨. 자재코드는 유사 제품 참조용 |

화면 UX:

1. STEP 1 · 시나리오 선택 (기제품 / 신제품)
2. STEP 2 · 자재코드 검색·선택 + 수량 입력
3. STEP 3 · (B 인 경우) 신제품 메타(금형 소재/난이도/크기 등) 추가 입력
4. STEP 4 · 예상 납기일 응답 확인 → 수주 확정

## 예상 납기일 로직 (방향성 확정 — 2026-05-07)

> 출처: [[Meetings/2026-05-07 - 수주 확정 후 프로세스 설명]] · [[Meetings/2026-04-30 - 클라이언트 답변 - 수주 시나리오 · 납기 로직]]

### 알고리즘 방향성 (2026-05-07 합의)

> [!important] 결정
> **알고리즘 본체 = ERP 기수주 데이터 기반 점진 추가 방식** (실시간 설비 연동·고정밀 예측 보류).

**현재 운영 (Before)**:
- 기본 약 **40일** 기준 + 생산관리 팀장 경험 기반 공정별 일수 수기 산정
- 매일 아침 누적 수주 보면서 엑셀에 공정별 일수 기입 (예: 사출 +7, 코팅 +3, 인쇄 +4)
- 외주처에 카톡으로 송수신

**시스템 목표**:
- 수주 입력 즉시 산출 (외주처 답변 기다리지 않음 — 판가/수주 가능성 빠른 판단)
- 예시: SKU A 기본 30일인데 같은 SKU 기수주 1주치 누적 → +7일 늦은 납기 제시
- SKU 단위 분석이 어렵다면 **외주처 단위 평균 처리일** 분석으로 대체

### 신제품 (금형 개발)

- **평균**: 약 **45일**
- **변동 변수** (영향순): 업체별 capa(1순위) > 금형 난이도 > 금형 소재 > 금형 크기

### 시스템 전제 (2026-05-07 갱신)

- **외주업체 생산계획 직접 수신 불가** → 시스템 내 **기수주 점유율** 기반으로 일정 지연 추정
- ERP 에 **공정별 납기 이력 데이터 존재 가능** (1년치 = 대용량) → 외주처 단위 평균 처리일 추정 가능. **존재 여부 확인 = Action Item**
- 데이터 출처/마스터 (2026-05-07 미팅에서 단순화):
  - **SKU-BOM-외주처 매핑 = ERP 에서 끌어옴** (별도 마스터 구축 X)
  - **공정 마스터 = 5단계** (사출 / 코팅 / 인쇄 / 박 / 조립) ([[01 - 사전 준비 - 등장인물 · 용어집#공정 5단계 (창신 화장품 용기)]])
  - **외주처 단위 평균 처리일** = ERP 이력 분석으로 산출
  - **내부 생산 라인 = 외주처와 동일 entity 로 처리** (개발 단순화)

### SKU 구조 주의 (2026-05-07 명시)

- **SKU = 공병 단위**, 제품과 1:1 아님
- 같은 SKU 라도 코팅·인쇄·박 후공정에 따라 **실제 제품 수십 개로 분기**
- BOM 분해: 외용기 / 내용기 / 외캡 / 내캡 / 리드

---

> [!warning] 개발 시 반드시 체크
> - SKU 마스터 데이터 클렌징이 **선결 조건**
> - 초기엔 실시간 피딩 없이 **배치 기반**으로 시작
> - 데이터 축적 후 ML 기반 고도화는 2단계로 분리
> - 외주처 기입 권한/책임 부서 확정 필요
