# Module 4 · 구매 · 정산

> 상위 문서: [[00 - AX-2 쉬운 설명서 (Index)]]

> [!summary] 한 줄로 말하면
> 수주/납품/공제 내역을 시스템이 자동 집계해 **외주처 · 생산관리 · 구매팀 3자가 컨펌만 하면 정산이 마무리**.

- **관련 주체**: 구매팀 · 생산관리팀 · 외주처
- **핵심 요소**: BOM 기반 정산 / 공제 로직(품질/재생산) / 3자 컨펌 프로세스

---

## 실제 현장 장면 (Before)

- 생산관리팀이 수기로 발주 내역을 구매팀에 전달하고 정산 요청.
- 거래명세표 누락/체크 누락으로 "완료 처리" 못 한 건이 월 5~6회.
- 재생산 공제/품질 귀책 공제 로직이 문서로 흩어져 있고 담당자마다 해석 다름.
- 구매팀 정산 건당 10~20분이지만, 추적·확인 시간 포함 시 훨씬 커짐.

## AX-2 적용 후 (After)

- 시스템이 BOM + 수주 + 납품 + 공제 내역을 자동 계산.
- 외주처/생산관리/구매 3자가 순차 컨펌만 하면 정산 완료.
- 거래명세표 업로드 없이는 정산 단계 진입 불가 → 누락 원천 차단.
- 공제 내역은 품질/재생산 이벤트와 자동 연결 (품질/구매팀 컨펌 로그 보존).

---

## 개발팀이 구현해야 할 기능

- [ ] (10)/(11) 발주·정산·공제 자동 계산 엔진 — BOM 기반
- [ ] 공제 로직: 재생산 사유 · 품질 귀책 · 수량 차이 케이스 분기
- [ ] 3자 컨펌 워크플로우 (외주처 → 생관 → 구매) 및 변경이력 감사 로그
- [ ] 거래명세표 업로드 게이트: 미등록 시 정산 단계 블록
- [ ] 정산 완료 건의 월간/건별 대시보드 — 누락 모니터링

---

> [!warning] 개발 시 반드시 체크
> - 상세 발주/정산/공제 로직은 실무 확인 후 스펙 확정
> - 품질 귀책 판단 주체(품질팀) 인풋 표준화 필요
> - 기존 ERP와의 이중 입력 방지 — **단일 소스 오브 트루스** 정의
> - 거래명세표 포맷/시점 정의 (외주 직납 케이스 포함)
