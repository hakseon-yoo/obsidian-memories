## 함께 일하는 팀 (등장인물)

| 팀/주체 | 역할 |
|--------|------|
| **영업팀** | 주문을 따와서 시스템에 입력 |
| **생산관리팀** | "컨트롤 타워" · 일정 짜고 외주에 뿌리고 확인 |
| **외주처** | 실제로 물건을 만드는 협력 업체 (A, B, C…) |
| **품질팀** | 불량 여부 판정 · Go/No Go 결정 |
| **자재팀** | 물류 · 배차 · 창고 예약 담당 |
| **구매팀** | 돈 정산 · 발주 · 공제 |
| **금형관리팀** | 금형 상태와 메인터넌스 관리 |
| **ODM / 고객사** | 완제품을 납품받는 최종 업체 |

## 꼭 알아야 할 용어

> [!info] 용어집
> - **수주 (受注)**: "주문 받기" — 고객사가 물건 만들어달라고 주문한 것
> - **SKU**: 창신은 **공병(공기용기) 단위**로 SKU 부여 (제품과 1:1 아님). 같은 SKU 라도 코팅·인쇄·박 후공정에 따라 실제 제품 수십 개로 분기 (2026-05-07 미팅 명시)
> - **BOM**: Bill of Materials — 완제품 1개 만들 때 필요한 부품 · 재료 리스트. 창신 BOM 분해 = **외용기 / 내용기 / 외캡 / 내캡 / 리드** 등
> - **WIP**: Work In Progress — 만드는 중인 반제품 (완성되지 않은 상태)
> - **납기일**: "언제까지 줄게요" 약속한 날짜
> - **외주처**: 일부 공정을 대신 만들어 주는 협력 업체
> - **T0 / T1 / T2**: 신제품 개발 단계 (0차 · 1차 · 2차 시제품)
> - **공제**: 정산할 때 제품 불량/재생산 이유로 금액을 깎는 것
> - **ERP**: 회사 업무 관리 시스템 (주문 · 재고 · 발주 등). 창신은 ERP 에 SKU-BOM-외주처 매핑 + 공정별 납기 이력 데이터 보유 (2026-05-07 미팅 확인)
> - **PDA**: 손에 들고 다니는 바코드/QR 스캐너 (창신 외주처에선 활용 불가)
> - **너징(Nudging)**: 외주처/생산팀이 컨펌·조정 회신 안 할 때 시스템이 자동 독촉하는 기능 (AX-2 핵심 가치)

## 공정 5단계 (창신 화장품 용기)

> 출처: [[Meetings/2026-05-07 - 수주 확정 후 프로세스 설명]]

| # | 공정 | 설명 |
|---|------|------|
| 1 | **사출** | 색이 입혀지지 않은 공병 제작 |
| 2 | **코팅** | 표면 마감 |
| 3 | **인쇄** | 재활용 정보 / 성분 표시 |
| 4 | **박** | 금박 / 은박 후가공 |
| 5 | **조립** | 뚜껑 / 내용기 / 외용기 등 파츠 조립 |

> [!warning] 공정 분산 구조
> - 한 업체가 모든 공정 담당하는 경우 거의 없음
> - SKU 별로 공정 흐름이 매우 다양 — **고정 패턴 X**
> - 외부 공정 후 **내부로 다시 돌아와 조립** 후 출하하는 케이스도 흔함
> - 내부 생산 라인도 모든 공정을 다 하지 않음 → 시스템 관점에서는 **내부 생산 = 외주처와 동일 로직**으로 처리 (2026-05-07 결정)
