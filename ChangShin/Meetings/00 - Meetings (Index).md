> [!abstract] 이 폴더의 쓰임
> 클라이언트(창신·아크 파트너스 등) · 사내 · 파트너 미팅 결과를 정리해 보존.

---

## 회의록 작성 규칙

### 파일명

```
YYYY-MM-DD - 핵심 주제.md
```

예: `2026-04-29 - 화장품 용기 데이터 프로젝트 범위 조정.md`

### 프론트매터 (필수)

```yaml
---
title: <회의 제목>
date: YYYY-MM-DD
project: 창신
attendees:                 # 참석자 (역할이 있으면 표기)
  - 이름 (역할/소속)
  - …
related:                   # 관련 문서 (선택)
  - "[[…]]"
tags:
  - meeting
  - <주제 태그>
---
```

### 본문 구조 (권장)

1. **TL;DR** — `> [!abstract]` 콜아웃 1~3줄 요약
2. **핵심 결정 (Critical Outcomes)** — 닫힌 결정 / 향후 영향이 큰 결정
3. **논의 내용 (Discussion)** — 주제별 섹션. 발언 전체 받아쓰기보다는 **결정·맥락·근거** 중심으로
4. **Action Items** — 표 (담당자 / 항목 / 기한 / 상태). [[00 - Action Board]] 와 동기화
5. **본인 영향 (Personal Takeaways)** — 본인 책임 영역에 미치는 항목 별도 표기 (선택)

### 동기화 규칙

회의록을 작성한 뒤 다음 갱신을 잊지 말 것:

- **새 액션** → [[00 - Action Board]] 의 적절한 카테고리에 추가 (출처: 회의록 위키링크)
- **새 결정** → [[Infrastructure/31 - Decision Log]] 에 `D-###` 추가 (관련 회의 링크)
- **클라이언트 답변** → [[AX-2 지능형 스케줄러/10 - 프로젝트 착수 질의 리스트]] 답변 칸 갱신
- **인프라/도메인 변경** → [[Infrastructure/32 - Deployment State]]
