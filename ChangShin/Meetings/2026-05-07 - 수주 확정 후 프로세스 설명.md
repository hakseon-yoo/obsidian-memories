## 3. 논의 내용

### 3-1. ERP 연동 (ERP-1 ~ ERP-5 답변)

| 질문 | 답변 |
|------|------|
| **ERP-1** 현재 ERP 정체 (제품·버전·벤더) | ❌ 미답변 — RFI 작성 후 확인 |
| **ERP-2** 연동 방식 가능성 (REST/DB/배치/큐) | ⏳ "ERP 자체 연동은 일반적으로 가능" — 구체 스펙은 ERP 접근 후 확인 |
| **ERP-3** 시스템이 ERP 에 직접 쓰기 가능한가? 사용자 이중 입력? | ⏳ "수주 확정 정보를 ERP 에 등록하는 구조" 언급 → 양방향 가능 추정. 구체 스펙은 후속 |
| **ERP-4** 창신 측 ERP 담당자 배정 | ⏳ **엄영태 팀장 계정으로 이미 발급된 것으로 클라 인지**. 본인은 받지 못함 → 회장님께 전달 재요청 약속 |
| **ERP-5** 실시간성 (폴링/푸시) | ✅ "실시간성 어렵다 = 생산 라인 → ERP 자동 입력 X" 의 의미. ERP 자체와의 송수신은 폴링/배치로 가능 |

→ ERP 연동은 모든 모듈의 **선결 과제**. 접근권 확보 → 데이터 탐색 → 연동 스펙 확정 순서.

### 3-2. 예상 납기 로직 (M1)

**현재 운영**:
- 기본 약 **40일** 기준 + 생산관리 팀장의 경험 기반 공정별 소요일 수기 산정
- 매일 아침 당일 누적 수주 정보를 보면서 엑셀에 공정별 일수 기입 (예: 사출 +7일, 코팅 +3일, 인쇄 +4일)
- 외주처에 카톡으로 엑셀 전달 → 회신 받아 수정 반영

**시스템 목표 (이번 미팅에서 합의)**:
- 수주 입력 즉시 예상 납기 자동 산출
- 외주처 답변 기다리는 방식은 의미 약함 — 판가/수주 가능성 판단을 빠르게 해야 함
- **알고리즘 본체 = ERP 기수주 데이터 기반 점진 추가** (예: SKU A 기본 30일인데 기수주 1주치 누적이면 +7일 늦은 납기 제시)

**데이터 활용 가능성**:
- ERP 에 **공정별 납기 소요 이력** 존재 가능 (1년치 = 대용량) — 클라 측이 확인 약속
- SKU 단위 분석이 어렵다면 **외주처 단위 평균 처리일** 분석 가능 ("이 외주처는 코팅 N개에 평균 며칠")

### 3-3. 스케줄 자동 배포 흐름 (M2)

- **시스템이 자동으로** SKU → 공정 순서·업체 파악 → 각 업체에 시점·완료기한 통지 (PPT (5))
- 예시: "1번 업체: 일주일 후 완료 가능합니까?" / "2번 업체: 일주일 후 물량 들어오는데 그 후 3일 가능?"
- 외주처 컨펌/조정 회신 (PPT (6)/(7))
- **응답 안 하면 시스템 자동 너징 (지속 알림)** — 핵심 가치
- **초기 운영 보완책**: 시스템 초창기엔 생산관리팀이 자동 산정 결과 한번 검토 후 외주처 발송 → 신뢰도 쌓이면 직접 발송
- **배치 전송 vs 건별 전송**: 미정 (추후 결정)
- **외주처 화면**: 신규 건만 보는 게 아니라 **기존 기수주 건까지 함께 조회 가능** (전체 맥락 파악 후 응답)

### 3-4. 생산 관제 / 품질 판정 (M3)

- **매일 확정 생산 계획 통지** (일 단위) — PPT (13)
- 현장 응답 두 유형:
  1. 확정 계획 확인 후 그대로 가능
  2. 처음엔 가능했지만 실제 진행 시 불가능 → 조정 요청
- **재조정 루프 = (5)~(10) 단계 반복** 구조 인정 (PPT (17))
- **품질 판정 = WIP 도 대상** — 외주처 / 내부 모두 시스템 내에서 Go/Stop 요청 가능

### 3-5. 물류 모듈 4종 슬롯 (M4)

미팅에서 정리된 4종 (PPT 5종을 단순화):
1. 내부 생산품 → 고객사 직배송
2. 내부 생산품 (WIP) → 외주처
3. 외주처 (WIP) → 내부 반입
4. 외주처 생산품 → 고객사 직납

- 자재팀: 배차 정보 + 상차 사양 입력
- PDA 태깅: 실제 상차 정보와 입력 정보 일치 검증
- 불일치 시 경고 / 알림

### 3-6. 도메인 이해 — 공정 5단계 + SKU/BOM 구조

**공정 5단계** (PPT 의 사출/후가공/도장/조립 보다 세분):
1. **사출** — 색 입혀지지 않은 공병 제작
2. **코팅** — 표면 마감
3. **인쇄** — 재활용 정보, 성분 표시
4. **박** — 금박/은박 후가공
5. **조립** — 뚜껑·내용기·외용기 등 파츠 조립

**공정 분산 구조**:
- 한 업체가 모든 공정 담당하는 경우 거의 없음
- SKU 별로 공정 흐름이 매우 다양 — **고정 패턴 X, 제품별 상이**
- 외부 공정 후 내부로 돌아와 조립 후 출하하는 케이스도 흔함

**SKU 구조 특수성**:
- **SKU = 공병 단위** (제품과 1:1 매핑 아님)
- 같은 SKU 라도 코팅/인쇄/박 후공정에 따라 **실제 제품 수십 개로 분기**
- 부품 레벨 분해: **외용기 / 내용기 / 외캡 / 내캡 / 리드** 등으로 다시 나뉨
- ERP 확인 시 이 구조 이해 필요 — 막히면 클라 측에 즉시 문의 가능
