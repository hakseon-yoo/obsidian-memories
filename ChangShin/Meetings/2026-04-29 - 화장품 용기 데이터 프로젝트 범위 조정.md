# 화장품 용기 데이터 프로젝트 범위 조정

> 상위 문서: [[00 - Meetings (Index)]]

> [!abstract] TL;DR
> 1차(5월 말) 범위는 **전체 구현이 아닌 일부 브랜드·카테고리·채널 기반 리포트**로 축소. 우선 브랜드 3개는 **달바·메디큐브·닥터 멜락신**. 모호한 "메가 트렌드" 섹션은 **축소/제외**, 대신 **판매 랭킹·용기 타입 분석**을 살린다. **로그인+마스터 계정** 필수, **AX1·AX2·AX3는 별도 URL** 구조로.

## 2. 논의 내용 (Discussion)

### 2-1. 프로젝트 출발점 — 감(感) → 데이터 기반 의사결정

**프리몰드 개념**
- 화장품 용기에서 특정 고객 전용이 아니라 **회사 자산으로 미리 만들어 둔 금형/용기**
- 색상·로고·후가공만 달리해 여러 고객사에 판매 가능
- **어떤 프리몰드를 개발해야 할지** 결정하는 것이 핵심 과제

**기존 의사결정 방식**
- 감과 영업 정보 중심 ("요즘 이런 게 잘 팔리는 것 같다")
- A/B/C 고객 요청 기반으로 개발 방향 결정

**데이터 프로젝트의 원래 목적**
- 백데이터 기반으로 개발 우선순위 결정
- 자사 SKU와 매칭되지 않는 시장 수요 발견
- 활용 갈래: ① 개발용 의사결정 데이터 / ② 영업용 트렌드·브랜드 리포트

### 2-2. 1차 범위 축소 — A부터 Z까지 한 번에 X

| 항목 | 1차 (5월 말) | 이후 |
|------|-------------|------|
| 카테고리 | 약 3개 | 확장 |
| 브랜드 | 약 3개 (달바·메디큐브·닥터 멜락신) | 확장 |
| SKU 데이터 관리·분석 | 일부 | 검색·매칭 고도화 |
| 리포트 생성 | 1차 목표 도달점 | — |
| 기능 (3D / 믹스 매칭 / NLP) | 미포함 | 단계적 |

**주의**: 5월 말 = 일부 범위 / 6월 말 MVP — 본인 [[AX-2 지능형 스케줄러/10 - 프로젝트 착수 질의 리스트#0. P0 빠른 참조 (착수 전 결정 필수)|P0 빠른 참조]] 의 "MVP 6월 말" 과 트랙 차이 가능성 — 추가 조율 필요. 회의에서도 비용 범위·확장 일정은 미확정으로 남음.

**조직 역할**
- **팀 스파르타 빌더스** = 실제 개발 수행
- **팀 스파르타** = 영업 측

### 2-3. 브랜드 리포트 우선순위

본부장이 가장 우선적으로 보고 싶어 하는 결과물 = 브랜드 리포트.

- 1차 대상: **달바 / 메디큐브 / 닥터 멜락신**
- 카테고리는 추가 결정 필요 (1개부터)

### 2-4. 카테고리 체계 — 대분류 ↔ 소분류 계층 구조

처음에 "카테고리" 의미 혼선 (기능 분류? 시장 분류? 브랜드 분석용 분류?) 발생. 합의:

- **대분류** 예: 스킨케어 / 메이크업 베이스 / 포인트 메이크업
- **소분류** 예: 세럼 / 앰플 / 로션 / 크림 / 파운데이션 / 립
- 대분류 클릭 → 전체 결과 / 소분류 클릭 → 상세 결과
- **채널마다 카테고리 체계 다를 수 있음** 인지
- 카테고리 분류안은 **고객사 측에서 정리·전달** 예정 (올리브영 분류 기준 권장)

### 2-5. 메가 트렌드 화면 — 핵심 문제: "트렌드" 정의 불명확

**문제**
- "트렌드 흐름" 개념이 개발자/실무자 모두에게 모호
- "트렌드 키워드를 어떤 조건으로 어떻게 뽑을지" 불명확 → 구현 어려움
- **시장 전체 트렌드 분석** vs **특정 브랜드 분석** 은 완전히 다른 문제

**구분 명확화**
- **시장 트렌드 분석** = 무엇을 개발할지 찾기 위한 출발점
- **브랜드 분석** = 특정 브랜드에 맞는 용기 제안

→ 메가 트렌드 화면은 시장용. 브랜드 리포트는 별도 시안 준비 중. 다음 주쯤 더 구체적 설명 가능.

### 2-6. 브랜드 분석의 진짜 목적 — 브랜드 에센스 추출

**핵심**
- 단순 판매 데이터보다 **용기 패키지 디자인 기준**을 읽어내는 것이 중요
- 분석 요소: **톤앤매너 / 필링·질감 계열 / 쉐입 / 브랜드 에센스 (메인 컨셉·캡 스타일 등)**

**브랜드 디자인 기준은 비교적 고정**
- 파스텔 톤 중심 / 특정 쉐입 일관 / 디자인 책임자·대표 기준
- 신제품이 나와도 본질적 에센스는 크게 변하지 않음

**활용 방식**
1. 특정 브랜드의 디자인 기준 도출
2. 자사 프리몰드/리몰드 자산 매칭
3. 매칭 자산 없으면 신규 개발 전략 수립

→ **브랜드 분석에서는 카테고리 중요도 상대적으로 낮음**. 핵심은 톤앤매너·쉐입·컨셉·에센스.

### 2-7. 시장 트렌드 → "잘 팔리는 제품" 으로 방향 전환

- 일반론적 시장 트렌드 자체는 시중 자료 많아 차별화 어려움
- 차별화 = **자사 영업과 제안에 바로 쓸 수 있는 데이터**

**대체 방향**
- "핫 트렌드 흐름" 같은 추상 표현 → "요즘 잘 팔리는 제품" / "플랫폼별 상위 랭킹"
- 랭킹 점수 산정 근거 명시 가능해야 (예: 91점이 왜 나왔는지) — 플랫폼별 비중·판매 지표·반영 비율
- 영업 활용 시나리오: 시장 1위 제품 → 카테고리별 이슈 제품 → 상위 제품에서 발견되는 용기 유형 (실리콘/스포이드/펌프 등) → 자사 용기 제안

### 2-8. 유지 vs 제외

**유지 (가치 큼)**
- 용기 타입 트렌드 추천
- 판매 데이터 기반 도출 용기 유형 지표
- 고객에게 어필되는 용기 시각화 지표
- 단, "트렌드"라는 텍스트는 혼란 → "타입별 추이" / "용기 타입 분포·추천" 등으로 재명명

**제외/축소**
- 클린 뷰티·저자극 등 기준 불명확한 선택형 섹션
- 메가 트렌드 직접 도출 시도

### 2-9. SKU 매핑과 5월 이후 분리

- 1차: 특정 브랜드·채널 데이터를 **자사 SKU 에 매핑** 후 리포트 작성
- 5월 이후: SKU 검색·매칭 고도화·3D·믹스 매칭·자연어 처리

**현재 본인이 진행한 SKU 매핑**: 카탈로그 PDF 기반 도메인 설계 + DB 시드 + 이미지 S3 ([[Infrastructure/31 - Decision Log#D-021|D-021]] / [[Infrastructure/31 - Decision Log#D-022|D-022]]) — 본 회의 SKU 매핑과 정합 가능성 점검 필요.

기존에 엄태 팀장·조 대표·아크 파트너스와 공유된 타임라인과 다를 수 있다는 우려 제기 → 정리 후 메일로 재공유 예정.

### 2-10. 접근 제어 · 웹 구조

- **로그인 페이지 필수** — 라이브러리 등 유출 민감 정보 포함 가능
- **마스터 계정** 기준 시작
- **고객 접근**은 일부 기능에 한해 제한적 (3D / 믹스 매치 / 향후 고객 접속 기능)
- 로그인 ID/PW · 비로그인 페이지 정책은 추후 조정 가능
- **모바일 화면 필요성** 언급 (앱 다운로드용 + 모바일 사이즈 화면). 단 앱 자체는 현 범위 외

### 2-11. AX1·AX2·AX3 화면 구조 — 별도 URL

- 탭 X, **각각 별도 URL**
- 예: `AX1`, `AX2`, `AX3` 형태
- → [[Infrastructure/31 - Decision Log#D-019|D-019]] 후속의 "Ingress path 단순화" 결정에 직접 영향. **path 통합(`/api/*`) 보다는 path/서브도메인 분리 유지가 회의 결과와 정합**.
- 화면 공유 중 타이핑 미보임 이슈 → 카카오톡으로 예시 전달 예정

### 2-12. 초안 공유 · 3D 도면 · NDA

- 브랜드 리포트 초안 작성 중 → 완성 후 공유
- 3D 도면 (캐드 기반) 보유 → 전달 가능
- **단, NDA 체결 (팀 스파르타 ↔ 창신) 선행 조건**

### 2-13. 마무리 합의

- 스파르타 측: 1차 진행 범위 정리해 회신 (금주 내)
- 고객사 측: 본부장 합의 후 카테고리 분류안 + 1차 카테고리 후보 전달
